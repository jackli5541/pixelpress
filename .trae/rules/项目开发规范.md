# 大模型自主开发规则规范

## 1. 目标
本规范用于约束大模型在 `pixelpress` 项目中的自主分析、设计、编码、重构、测试和文档更新行为，确保其产出满足以下目标：

+ 不破坏既有系统契约
+ 不绕过状态机和版本控制
+ 不引入不可回溯的隐式副作用
+ 不牺牲印刷一致性、可解释性和可恢复性
+ 不在缺少明确契约的前提下擅自扩展核心能力

本规范优先级高于自由发挥，高于实现便捷性，高于局部性能优化。

## 2. 总体原则
### 2.1 先契约，后实现
任何新增或修改功能，必须先确认以下契约是否存在且一致：

+ `BookLayout`
+ 五层流水线输入输出契约
+ 前后端 API 契约
+ 状态机定义
+ 用户微调操作契约
+ 渲染/导出一致性契约

若契约不存在或不完整，大模型不得直接编码，应先补充契约文档或结构定义。

### 2.2 单一事实来源
以下对象必须分别作为唯一事实来源（Single Source of Truth）：

+ `BookLayout`：全书当前版本的唯一事实来源
+ `TaskState`：异步任务执行状态的唯一事实来源
+ `AlbumState`：相册项目业务阶段的唯一事实来源
+ `FeatureState`：照片特征提取状态的唯一事实来源

任何模块不得通过推断、缓存残留或前端局部状态替代这些对象。

### 2.3 可重放、可恢复、可审计
所有自动生成结果必须满足：

+ 可通过 `seed + pipeline_version + model_versions + input_hash` 重放
+ 可通过 checkpoint 从中断处恢复
+ 可通过审计日志回溯是谁、何时、基于哪个版本触发了什么变更

### 2.4 局部修改优先
用户微调和局部重排必须优先保持最小影响范围：

+ 改裁切：只影响当前页
+ 换图：只影响当前页和必要的相邻页
+ 改文案：只影响文本层，不影响几何布局
+ 改章节结构：允许触发页面规划及其下游重算
+ 改全局偏好（如主角人物、风格、开本）：允许触发全书重算

## 3. 强制状态机规范
### 3.1 不允许混用的状态
大模型不得将以下状态混为一谈：

+ `TaskState.status`：`queued | running | completed | failed | cancelled | timed_out | partial`
+ `BookLayout.status`：`draft | locked | exported | archived`
+ `FeatureState.status`：`pending | extracting | partial | ready | failed`
+ `AlbumState.status`：`draft | generating | reviewable | locked | ordered | archived`

### 3.2 合法状态迁移
仅允许以下状态迁移：

+ `FeatureState`：`pending -> extracting -> ready`
+ `FeatureState`：`pending/extracting -> partial`
+ `FeatureState`：`pending/extracting -> failed`
+ `TaskState`：`queued -> running -> completed`
+ `TaskState`：`queued/running -> failed`
+ `TaskState`：`queued/running -> cancelled`
+ `TaskState`：`running -> timed_out`
+ `TaskState`：`running -> partial`
+ `BookLayout`：`draft -> locked -> exported`
+ `BookLayout`：`draft -> archived`
+ `AlbumState`：`draft -> generating -> reviewable -> locked -> ordered`

禁止以下行为：

+ 从 `locked` 回退到 `draft` 而不生成新版本
+ 在 `exported` 后继续原地修改同一 `BookLayout.version`
+ 在 `FeatureState != ready/partial` 且无慢速通道标记时启动正式排版
+ 在 `TaskState != completed` 时将结果标记为可下单

### 3.3 版本规则
每次以下行为都必须创建 `BookLayout.version + 1`：

+ 全书重排
+ 用户微调后产生新的布局结果
+ 章节结构变化
+ 风格变化影响渲染结果
+ 导出前的任何结构性修订

仅在纯读取、纯评分重算且不改变布局内容时，允许复用当前版本号。

## 4. 并发与幂等规则
### 4.1 幂等键
所有会修改状态的接口必须支持幂等键：

+ 生成排版：`idempotency_key`
+ 用户微调：`operation_id`
+ 锁定版本：`lock_request_id`
+ 导出 PDF：`export_request_id`
+ 提交订单：`order_request_id`

若缺少幂等键，大模型不得实现该写接口为生产可用版本。

### 4.2 乐观锁
所有修改 `BookLayout` 的操作必须带上：

+ `album_id`
+ `base_version`
+ `expected_status`

若当前版本不等于 `base_version`，必须拒绝写入并返回冲突错误。

### 4.3 单相册单活跃生成任务
同一 `album_id` 在同一时刻只允许一个全书级生成任务处于 `queued/running`。

允许并行的只有：

+ 缩略图渲染
+ 剩余页异步渲染
+ 只读查询
+ 不改布局的评分分析

## 5. 五层流水线实现规则
### 5.1 节点输入输出规则
每个 LangGraph 节点必须：

+ 只读取显式输入对象
+ 输出结构化对象，不返回自由文本作为核心结果
+ 不直接修改数据库中的其他领域对象
+ 不跨层消费未声明的内部字段

### 5.2 节点副作用规则
以下副作用必须放在专门节点或适配器内，不得混进算法节点：

+ 数据库存取
+ 队列入队
+ 对象存储上传
+ 预览图渲染
+ PDF 导出
+ 外部订单调用

### 5.3 回退规则
评分失败时，必须依据 `repair_hints` 判断回退层级：

+ 版式局部问题 -> 回退到 `版式生成`
+ 章节节奏或页数预算问题 -> 回退到 `页面规划`
+ 章节边界明显错误 -> 回退到 `章节聚类`

不得在没有问题定位依据时一律回退到最上游。

## 6. 用户微调操作契约
### 6.1 禁止自由文本直改布局
用户操作进入系统后，必须先转为结构化 Patch，不得让大模型直接基于自然语言改 `BookLayout`。

### 6.2 标准操作集合
只允许以下结构化操作：

+ `replace_photo`
+ `swap_page_photos`
+ `adjust_crop`
+ `set_caption`
+ `set_chapter_title`
+ `mark_page_disliked`
+ `set_hero_person`
+ `merge_chapters`
+ `split_chapter`
+ `reorder_chapters`
+ `lock_layout`
+ `request_export`
+ `submit_order`

### 6.3 Patch 结构
所有写操作必须至少包含：

```json
{
  "operation_id": "string",
  "album_id": "string",
  "base_version": 3,
  "op": "replace_photo",
  "payload": {},
  "actor": {
    "type": "user | system | agent",
    "id": "string"
  }
}
```

### 6.4 操作影响范围
大模型必须先判断操作级别，再决定重算范围：

+ `adjust_crop` -> 当前页重渲染
+ `replace_photo` -> 当前页 + 相邻页局部重排
+ `set_caption` / `set_chapter_title` -> 仅文案层
+ `set_hero_person` -> 页面规划 + 评分重算，默认不得只改评分不改规划
+ `merge_chapters` / `split_chapter` / `reorder_chapters` -> 页面规划及其下游全量重算

## 7. 渲染与导出一致性规则
### 7.1 所见即所得
预览与 PDF 输出必须共享同一份 `BookLayout.version`，不得允许：

+ 预览来自版本 N，导出来自版本 N+1
+ 锁定后仍因字体替换、裁切重算导致导出结果变化
+ 前端本地样式覆盖服务端排版结果

### 7.2 强制记录的导出参数
导出时必须固化以下参数：

+ 页面尺寸
+ 出血尺寸
+ 安全区
+ 字体文件版本
+ 颜色空间策略
+ 图片采样与压缩策略
+ 页码顺序
+ 渲染引擎版本

### 7.3 渲染不可变性
一旦 `BookLayout.status = locked`，必须冻结：

+ 页面几何布局
+ 裁切框
+ 文本框坐标
+ 导出参数
+ 资源引用版本

## 8. 缓存与失效规则
### 8.1 缓存键组成
以下缓存必须显式带版本键：

+ 特征缓存键 = `photo_id + model_version`
+ 布局缓存键 = `album_id + photo_set_hash + style + book_size + template_version + scoring_version`
+ 缩略图缓存键 = `book_layout_version + render_version + page_id`
+ PDF 缓存键 = `book_layout_version + export_profile_version`

### 8.2 失效触发器
以下情况必须使相关缓存失效：

+ 照片替换或删除
+ 模型版本升级
+ 模板 DSL 升级
+ 评分权重升级
+ 字体或导出参数升级
+ 用户微调生成新布局版本

### 8.3 禁止的缓存行为
禁止：

+ 用旧模型特征直接参与新版本排版而不打过期标记
+ 用旧布局缩略图冒充新版本预览
+ 仅按 `album_id` 读取布局缓存而忽略版本号

## 9. 错误处理与降级规则
### 9.1 降级不等于成功
若系统返回部分结果，必须显式标记：

+ `task_status = partial`
+ `book_layout.is_partial = true`
+ `degrade_reasons = []`
+ `allow_export = false`
+ `allow_order = false`

### 9.2 可恢复性
任何降级结果都必须支持：

+ 用户继续等待
+ 用户重新触发完整版流程
+ 用户接受仅预览用途结果

### 9.3 不可静默吞错
大模型不得在以下情况下静默继续：

+ 模型服务不可用
+ 关键字段缺失
+ 状态迁移非法
+ 版本冲突
+ 导出参数不完整

## 10. 技术栈与边界规则
### 10.1 跨语言边界先定义再集成
若采用 `FastAPI + BullMQ`，必须先定义清楚：

+ 谁负责入队
+ 谁负责消费
+ 任务 payload 的 JSON Schema
+ 任务状态回写协议
+ 重试和死信策略

若这些边界未定义，大模型不得默认将 Python 服务与 Node 队列直接耦合实现。

### 10.2 优先低耦合适配层
跨服务通信必须通过：

+ 明确的 DTO
+ JSON Schema / Pydantic 模型
+ 明确的超时与重试策略

不得依赖隐式字段、弱约定命名或运行时猜测。

## 11. 测试与验收规则
### 11.1 必测范围
以下改动必须带测试或可复现实验：

+ 状态机变更
+ `BookLayout` 字段变更
+ 五层输入输出契约变更
+ 微调 Patch 语义变更
+ 缓存键策略变更
+ PDF 导出参数变更

### 11.2 最低测试集
每次提交至少覆盖：

+ 1 个成功主流程用例
+ 1 个版本冲突用例
+ 1 个非法状态迁移用例
+ 1 个局部重排范围验证用例
+ 1 个降级结果不可下单用例

### 11.3 禁止的提交
以下情况不得提交为“完成”：

+ 只改实现，不更新契约
+ 只改契约，不校验兼容性
+ 只跑 happy path，不覆盖异常路径
+ 引入新状态但无迁移图和守卫逻辑

## 12. 大模型执行流程规范
### 12.1 接到任务后的顺序
大模型执行任务时必须按以下顺序：

1. 识别任务影响的领域对象、状态机和契约。
2. 判断是新增、修改还是修复。
3. 检查是否会影响版本、幂等、缓存、导出一致性。
4. 若契约缺失，先补契约再编码。
5. 实现最小变更。
6. 运行针对性验证。
7. 更新文档与审计说明。

### 12.2 输出要求
大模型在完成任务时，必须明确说明：

+ 修改了哪些契约
+ 修改了哪些状态迁移
+ 是否影响 `BookLayout.version`
+ 是否影响缓存键和失效逻辑
+ 是否影响预览/导出一致性
+ 如何验证

## 13. 项目当前补充约束
结合现有设计方案，新增以下强制约束：

+ `set_hero_person` 不得只改评分权重，必须至少触发页面规划层重新评估是否需要换图或调页序。
+ `partial` 结果不得进入 `locked`、`exported`、`ordered`。
+ `BookLayout.status` 不足以表达相册业务状态，必须补充 `AlbumState`。
+ `TaskState` 与 `BookLayout.status` 必须分离存储，不得共用单个 `status` 字段。
+ 任一生成、微调、锁定、导出、下单接口都必须支持幂等键和版本检查。
+ 渲染预览与 PDF 导出必须绑定同一 `BookLayout.version` 和同一导出参数快照。

## 14. 违规处理
若大模型生成的代码违反本规范，应视为未完成实现，必须回退到最近一个满足规范的契约点，重新执行：

+ 契约补全
+ 状态机修正
+ 版本控制修正
+ 测试补齐
+ 文档补齐

在规范与实现冲突时，以规范为准；在规范与业务目标冲突时，必须先升级规范，再修改实现。
