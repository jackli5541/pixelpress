## 二、Git 分支与提交规范

### 分支命名
```
feat/database-orm      feat/ai-prompt           feat/frontend-dnd
feat/file-storage       feat/ai-chapter          feat/frontend-modal
feat/user-auth          feat/ai-layout           feat/admin-tasks
```

### Commit 格式
```
<type>(<scope>): <subject>

例：
feat(db): PostgreSQL 接入，Album/Photo ORM 模型完成
feat(ai): 照片评估 Prompt 方案及验证脚本
fix(dnd): 修复拖拽后照片列表未刷新
refactor(storage): 抽象 FileStorage 接口
```

### PR 流程
1. 从 `main` 切分支
2. 开发 → 自测 → push
3. 提 PR，指定另一人 review
4. review 通过 → squash merge 到 main
